CMS - content management system

MVC
	In Model (Data Class) werden Daten "modelliert" - CRUD (create read update delete)
	Controller arbeitet allgemein nur mit den (öffentlichen) Methoden der Klassen/Objekte
		Erzeugt Objekte, löscht Objekte etc. (alles sehr Objektebezogen)
	Views präsentieren EIGENSCHAFTEN der Objekte (z.B. Bild ist so und so groß)
		


Warum Klassen?
	Internes Verhalten "kapseln"
	Erzeuge Objekt, lege es ab -> Klasse erledigt intern Dinge wie Speicherungsweise
	Bleibt Programmierern überlassen, wie Eigenschaften und Methoden realisiert werden
	UML-Klassendiagramme (unified modelling language)
		- private
		+ public
		# protected (kann nur an erbende und klar bestimmte andere Klassen weitergegeben werden)
	
Verweis von Objekt auf seine Eigenschaft über "->" (in java ".")
	z.B. $this->thumb statt this.thumb
